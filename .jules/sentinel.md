## 2024-05-20 - Path Traversal Vulnerability in Tool Filesystem Path Resolution
**Vulnerability:** The `_resolve_path` function in `src/bub/builtin/tools.py` allowed path traversal. It explicitly bypassed the workspace containment check for absolute paths (`if path.is_absolute(): return path`) and lacked proper checking for relative path directory traversal using `../` before operating on the resolved path. This allowed `fs.read`, `fs.write`, and `fs.edit` tools to access arbitrary files on the host system.
**Learning:** Checking for `.is_absolute()` is insufficient for containment if the path isn't securely re-based. Simply combining a base directory with an untrusted relative path using `Path(base) / path` does not prevent the resulting path from escaping the base directory if `path` contains `../`.
**Prevention:** Always use `Path.resolve()` on the combined path and then strictly enforce containment using `.is_relative_to(base_dir)`. When an absolute path is received from an agent, strip its root anchor (e.g., using `len(path.anchor)`) to treat it as relative to the configured workspace before resolution.
