## 2024-05-24 - Path Traversal Vulnerability in `_resolve_path` File Tool
**Vulnerability:** The `_resolve_path` helper in `src/bub/builtin/tools.py` allowed path traversal. User-provided absolute paths (like `/etc/passwd`) were directly returned or concatenated, allowing `fs.read`, `fs.write`, and `fs.edit` tools to access files outside the restricted agent workspace.
**Learning:** Checking for absolute paths and simply returning them or using `.resolve()` without verifying containment against the root workspace directory leaves the system vulnerable to absolute path traversal. Using `path.is_absolute()` is not enough to secure file access operations if containment is not validated afterward.
**Prevention:** Always re-base absolute paths to the workspace by stripping their root anchor (`len(path.anchor)`). Critically, always perform a final boundary check using `resolved_path.is_relative_to(workspace_path)` to ensure the canonical path strictly resides within the allowed directory tree.
